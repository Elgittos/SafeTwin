SafeTwin Local Windows Signing

Personal-use signing notes for Luis Chaname.

What Problem This Solves

Windows blocks unknown unsigned apps because it cannot tell who created the executable or whether it was modified after it was built. SafeTwin is your own app, but Windows still treats an unsigned installer like an unknown file.

The local signing setup gives your Windows user account a certificate that says: this SafeTwin build was signed by your local trusted certificate.

This is free and useful for your own machine. It is not the same as a paid public code-signing certificate.

What Was Added

  • scripts/sign-local-windows.ps1 creates or reuses a local SafeTwin code-signing certificate.
  • The certificate subject is CN=Luis Chaname SafeTwin Local Code Signing.
  • The certificate is stored in Cert:\CurrentUser\My.
  • The public certificate is trusted in Cert:\CurrentUser\Root.
  • The public certificate is trusted in Cert:\CurrentUser\TrustedPublisher.
  • Electron Forge runs the signing script during Windows builds.

What Gets Signed

During npm.cmd run make, the build now signs:

out\SafeTwin-win32-x64\SafeTwin.exe
out\make\squirrel.windows\x64\SafeTwinSetup.exe

How To Verify A Signature

Run this in PowerShell:

Get-AuthenticodeSignature "C:\Users\LuisDev\Desktop\SafeTwinSetup.exe" | Format-List

The important result is:

Status: Valid
SignerCertificate Subject: CN=Luis Chaname SafeTwin Local Code Signing

What This Does Not Solve

A local self-signed certificate only helps machines where that certificate has been trusted. It does not make the app automatically trusted by strangers on Reddit, GitHub, or another Windows computer.

For public trust, SafeTwin would need a public code-signing option such as Microsoft Azure Artifact Signing, Microsoft Store signing, or a traditional code-signing certificate.

Why This Is Still Useful

  • You do not need to pay just to run your own app.
  • Your local SafeTwin installer and app are no longer completely unsigned.
  • Future builds repeat the same signing step automatically.
  • You can still move to public signing later without changing the app architecture.

How To Remove The Local Trust Later

If you ever want to remove the local certificate, open PowerShell and inspect matching certificates:

Get-ChildItem Cert:\CurrentUser\My, Cert:\CurrentUser\Root, Cert:\CurrentUser\TrustedPublisher |
  Where-Object { $_.Subject -eq "CN=Luis Chaname SafeTwin Local Code Signing" }

Then remove the matching certificates carefully:

Get-ChildItem Cert:\CurrentUser\My, Cert:\CurrentUser\Root, Cert:\CurrentUser\TrustedPublisher |
  Where-Object { $_.Subject -eq "CN=Luis Chaname SafeTwin Local Code Signing" } |
  Remove-Item

Bottom Line

Local signing is the right free fix for personal use. Public signing is only needed when SafeTwin is ready for wider distribution to people who do not already trust your local certificate.
